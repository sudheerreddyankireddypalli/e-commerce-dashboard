# e-commerce-dashboard
This project is an E‑commerce Sales Analysis Dashboard built in Power BI.
# Problem
The dataset revealed a critical issue:
Sales were consistently high, but profit margins remained very low.  
This imbalance made it difficult to identify which products, cities, or payment methods were driving revenue versus profitability.
# Solution
I cleaned and transformed raw sales data using Power Query, then designed an interactive dashboard that:

Visualizes Total Sales vs. Total Profit side by side

Breaks down performance by product category, city, and payment method

Uses slicers and KPIs for dynamic filtering and real‑time insights

Highlights areas where revenue is strong but profitability is weak
# Key Features
📌 Category, City, and Product slicers

📈 KPI cards for revenue and profit tracking

🥧 Payment method analysis (Card, UPI, Net Banking, etc.)

🌍 City‑wise comparison of sales and profit

📊 Product‑wise sales breakdown
